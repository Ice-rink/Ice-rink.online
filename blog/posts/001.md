# 搭建属于你的 Minecraft 基岩版服务器

## 前言

想和小伙伴们一起在方块世界里快乐冒险，却受够了领域服的各种限制？或者想给服务器装上酷炫的插件，又不知道从何入手？这篇教程就是为你准备的！

我们将从零开始，在 Windows Server 环境下，基于 **LeviLamina**（以下简称 LL）搭建一个高性能的 Minecraft 基岩版专用服务器（BDS）。文末还会附上一份真实的服务器运维文档作为进阶参考，帮你从“搭起来”到“维护好”，一站式搞定。✨

> **小白友好声明**：本教程会尽量用通俗的语言解释每一步操作。遇到不懂的名词别慌张，跟着走就行～如果中途卡住了，欢迎回来评论区留言，我会尽量帮你解答！

## 准备工作

在动手之前，我们需要准备好几样东西：

### 1. 一台服务器（或电脑）
- **云服务器**：推荐腾讯云、阿里云等轻量应用服务器，系统选择 **Windows Server 2022**（本文以该系统为例）。配置建议 2 核 4G 起步，带宽按需选择。
- **自己电脑**：如果只是和小伙伴联机测试，用自己电脑开服也行，但需要保持电脑一直开机，且网络要允许他人连接。配置方面，基岩版服务器对 CPU 单核性能要求较高，内存建议预留 2G 以上给服务端使用。

### 2. 基岩版专用服务端（BDS）
从 Minecraft 官方下载基岩版服务端压缩包，解压后会看到一个 `bedrock_server.exe`，这就是服务端的核心程序。不过先别急着运行，我们接下来要给本体装上一个强大的“引擎”——LeviLamina。

### 3. LeviLamina（LL）
LL 是 BDS 的第三方插件加载器，相当于给服务器装上了 mod 平台。有了它，才能运行各种插件，让服务器变得丰富多彩。你需要从 LeviLamina 的官方 GitHub 仓库下载对应版本的安装包。

## 步骤一：安装 BDS 与 LeviLamina

1. **安装 BDS**  
   将官方服务端压缩包解压到你要开服的文件夹（比如 `C:\MCServer\`），确保解压后能看到 `bedrock_server.exe` 和一系列 DLL 文件。双击运行一次 `bedrock_server.exe`，它会自动生成世界文件夹、`server.properties` 配置文件以及权限文件等。看到命令行窗口出现“Server started”字样后，输入 `stop` 回车关闭服务端，第一次初始化就完成了。

2. **安装 LeviLamina**  
   从 LL 官方仓库下载 `LeviLamina-Windows-x64.zip`，将解压出的所有文件**覆盖**到 BDS 的根目录（和 `bedrock_server.exe` 同一文件夹）。注意 LL 版本必须和你下载的 BDS 版本严格匹配，比如 BDS 1.21.93 需要搭配 LL 1.4.3，版本不对会导致启动失败。覆盖完成后，运行 LL 附带的 `PeEditor.exe` 或双击一次 `bedrock_server.exe` 完成补丁应用。

> 🔧 小提示：如果安装成功，服务器启动后会看到 LL 的加载信息，同时根目录下会多出一个 `plugins` 文件夹，这就是你未来放置插件的地方。

## 步骤二：基础配置调优

在 `server.properties` 文件中，你可以调整以下关键参数（用记事本打开即可）：

| 配置项 | 推荐值 | 说明 |
|---|---|---|
| `server-name` | 你的服务器名字 | 玩家在服务器列表中看到的名称 |
| `max-players` | 10 ~ 50 | 根据服务器配置调整，家庭宽带建议 10 人以内 |
| `view-distance` | 6 ~ 8 | 视距，数值越低越流畅，对 CPU 影响很大 |
| `tick-distance` | 4 | 区块加载范围，同样影响性能，建议比 view-distance 小 2 |
| `online-mode` | true | 正版验证，建议开启防熊孩子 |
| `allow-cheats` | false | 是否允许作弊，生存服请关掉 |
| `default-permission-level` | member | 新玩家的默认权限等级 |

保存后关闭文件。如果有懂网络的小伙伴，别忘了在系统防火墙和云服务器安全组中放行 **19132** 端口（UDP 协议），这是基岩版默认的通信端口，不通的话外网玩家会连不上。

## 步骤三：安装插件

LL 生态下的插件通常以 `.dll` 或脚本形式存放。把下载好的插件文件丢进 `plugins` 文件夹，重启服务器即可自动加载。

作为起步，可以参考这套经典插件组合（来自 QYServer 实际运维文档）：

- **基础优化**：`LeviOptimize`（性能优化）、`Cleaner`（清理掉落物）、`FixRakNetBug`（修复网络漏洞）
- **安全防护**：`LeviAntiCheat`（反作弊）、`AntiXray`（防透视挖矿）
- **管理工具**：`GMEssentials`（基础指令集）、`BackupHelper`（自动备份）、`UpdateNotice`（更新提醒）
- **趣味功能**：`DynamicLights`（动态光源）、`FastMiner`（连锁挖矿）、`ShoppingMall`（玩家商店）

> 🧩 插件不是越多越好，每个插件都会消耗少量性能。建议从 5-10 个核心插件起步，根据实际需求逐步增加，装一个测一个，避免一次性塞太多导致启动报错找不到原因。

## 进阶：服务器运维的日常

服务端跑起来了，但“开服一时爽，维护火葬场”是每个腐竹的必经之路。下面是来自 QYServer 运维文档的精简版日常清单，帮你少走弯路：

### 1. 监控三件套
- **TPS / MSPT**：在控制台输入 `/sinfo`（LL 插件提供）查看。TPS 应稳定在 20，MSPT 越低越好。如果 MSPT 持续超过 50ms（即 TPS 开始明显掉下 20），就要检查是否插件冲突、实体太多或有人在跑大型机械。
- **CPU / 内存**：任务管理器盯紧，CPU 长时间 90%+ 就该升级配置或优化视距了；内存注意观察是否有持续增长不释放的情况，可能是某个插件的内存泄漏。
- **磁盘空间**：定期看看剩余空间，特别是开启了自动备份后，备份文件会越攒越多。

### 2. 日志巡视
重点关注 `Logs` 文件夹下的日志文件：
- **聊天日志**：看有没有人吵架、发广告，及时调节或封禁。
- **报错日志**：出现 `Error` 或 `Exception` 字样的行，通常会指出哪个插件出了问题。截图保存后再尝试修复或暂时禁用相关插件。
- **警告日志**：`Warning` 级别的问题暂时不影响运行，但放任不管将来可能演变成严重故障，建议定期清理掉能解决的警告项。

### 3. 故障定级与应对
参考 QYServer 的故障分级标准，心里先有个底：

| 级别 | 表现 | 响应要求 |
|---|---|---|
| P0 致命 | 无法启动、无限崩溃、玩家数据丢失 | 立即处理，7×24小时应急 |
| P1 严重 | 外网无法登录、核心功能挂掉 | 10-30分钟内响应 |
| P2 一般 | 网络卡顿、非关键报错 | 几小时内处理 |
| P3 轻微 | 文字错误、界面错位 | 下次更新再修 |
| P4 建议 | 优化类需求 | 排期处理即可 |

### 4. 备份铁律
- 每天至少一次世界和插件配置的全量备份，`BackupHelper` 插件可以自动完成这件事。
- 做任何大改动前（换插件、改配置、升级版本），**手动备份一次当前的整个文件夹**，出事了可以直接回滚。千万别对自己的操作过于自信，这是无数腐竹用血泪换来的教训。

## 真实案例参考

如果你好奇一个成熟的 LL 服务器长什么样，下面是 QYServer 的生产环境配置，供你参考：

- **操作系统**：Windows Server 2022 Datacenter
- **硬件配置**：Intel Xeon E3-1270 v3（8 核 @ 3.50GHz）+ 11 GB RAM
- **服务端版本**：BDS 1.21.93 + LeviLamina 1.4.3
- **已安装 32 个插件**，涵盖了反作弊、经济系统、成就、动态光源、土地保护等多种功能

这表明，在主流云服务器配置下，一台 LL 服务器可以稳定承载大量插件的运行。你的服务器不需要一步到位装这么多，先挑几个最需要的插件跑起来，等熟悉了再慢慢扩展就好。
